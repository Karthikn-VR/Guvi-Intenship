ABOUT THIS PROJECT

This retro arcade–styled landing experience is built with a strict black-and-white pixel aesthetic. It uses a fixed, transparent navbar; pixel typography; strong borders; and offset shadows to create depth while staying minimal.

Design Principles

Monochrome palette; Press Start 2P for headings and Inter for body; thick black borders; 6px offset pixel shadows; minimal layout; pixel accents outside boxes only.

Backend Overview

The authentication and dashboard features use lightweight PHP endpoints with JSON responses. Client-side JavaScript (jQuery) sends AJAX requests to these endpoints for login, registration, profile retrieval/update, and dashboard data. A demo mode exists that stores test user data in localStorage without server calls.

Homepage Structure

Navbar with GUVI logo and tagline plus Explore, About, Login. Glass-style hero card with title/description and two CTAs. Right-side feature cards for Web, AI, and Cloud themes. Background visible on the right for balance.

Interactions & Routing

Explore opens the official GUVI platform; Login/Register goes to the auth page; About provides this reviewer-focused documentation. Buttons have retro hover-lift and pixel shadow animation.

Responsiveness

On smaller screens, hero centers and feature cards stack below. Buttons stack vertically, and background positioning adjusts for a balanced presentation.

Styling System

All homepage styles live in css/retro.css. Components include navbar, hero card, feature cards, pixel decorations, and responsive grids. About page uses distinct classes to avoid the homepage glass hero.

Extensibility

New pages can reuse pixel border/shadow patterns and typography. Feature sections can be added without breaking the aesthetic. Pixel density is tunable and kept outside content boxes.

PHP Endpoints

login.php: Validates credentials, returns a session token and username on success. register.php: Creates a new user record with basic profile fields. profile.php: GET returns profile for a given username; POST updates age, DOB, and contact. dashboard.php: Returns a JSON payload with recommended role, skill score, top skills, recent activity, and a motivational message.

JavaScript Flow

Auth UI uses jQuery to handle form submissions and view toggles. Login/Register forms serialize inputs and send AJAX requests to the PHP endpoints. On success, the client stores a session token and username in localStorage and renders the dashboard. Demo mode (username/password “test”) bypasses the server and uses localStorage to simulate profile data.

Database Setup

A setup script initializes the database and tables. Credentials are configured in the setup file and should be secured in production (environment variables or server-side config). The application expects standard user tables for auth and profile fields. No credentials are displayed in this documentation.

Security Notes

Use HTTPS in production, hash passwords on the server, validate inputs on both client and server, and avoid storing sensitive information in localStorage. Session tokens should be short-lived and validated server-side.

Reviewer Notes

Key checks: consistent retro styling, clean routing, accessible typography, balanced layout. No sensitive credentials are exposed.
